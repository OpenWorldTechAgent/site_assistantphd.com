# Specification: MVP Website - assistantphd.com

## Overview
This track covers the development of the initial MVP for assistantphd.com, focusing on a modern, minimalist web presence that allows users to explore services and book consultations.

## Functional Requirements
### 1. Landing Page
- High-impact hero section with value proposition.
- Brief overview of core services.
- Clear Call-to-Action (CTA) for booking.

### 2. About Us
- Mission statement and brand story.
- Focus on expert, PhD-level precision.

### 3. Services Catalog
- List of services with titles and descriptions.
- "What's Included" checklist for each service.
- Direct link to booking or contact form.

### 4. Appointment Scheduling
- Integration with a third-party tool (e.g., Calendly).
- Support for Discovery Calls and Service Kickoffs.

### 5. Contact Form
- Basic info: Name, Email.
- Project details: Company, Website, Start Date.
- Service interest dropdown.

## Non-Functional Requirements
- **Performance:** Optimized assets for fast loading.
- **Responsiveness:** Fully functional on all device sizes.
- **SEO:** Metadata and semantic HTML for search visibility.

## Tech Stack
- Next.js, Tailwind CSS, PostgreSQL (for form submissions if needed), API Routes.
